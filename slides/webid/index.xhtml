Why have Universal Login for the Web?

  • Entering a username and password every time you have to log in to a different site is painful.
  • Specifying friends, contact information and preferences for each new social network is tedious.
  • Most passwords are not very secure because people use too many sites to remember a different password for each login.
  • Identity and Discovery is starting to become a big problem for the Web.

Identity on the Web

Web identity should provide affordances for...

  • Personal Control: One's identity should never be left to the whims of others.
  • Decentralization: The communication of human identity is fully decentralized by nature.
  • Access Control: One person has multiple identities which are often context sensitive.
  • Nuance: The identity that we choose is always affected, sometimes in very small or very large ways, by the person or audience receiving the message.

Not a replacement for authorization - OAuth is already good at that.

WebID Demo

A demonstration of the WebID technology.

  • Based on core Web concepts and standards: decentralized, RDF, Linked Data, HTML, RDFa, X509, TLS.
  • Pure Javascript and Flash implementation, using browser technology that exists today.
  • Simple, uniform UI experience across all browsers.
  • Try the demo

OpenID

  • WebID is backwards-compatible with OpenID and may be able to be expressed as an OpenID extension
  • OpenID isn't quite decentralized - requires the identity provider to be available at all times.
  • Complex attribute exchange protocol is not easily extensible (RDF is extensible).
  • More on OpenID shortcomings...

WebFinger

Focuses on profile discoverability, not authentication or a profile expression mechanism.

  • Requires strange XRD format, modification to DNS entries.
  • Works with WebID and OpenID.

/.well-known/

Directory for well known files such as robots.txt, avatar images, WebIDs, etc.

  • Generally agreed that it's not an ideal solution - bad for the Web.
  • Works well with WebID and OpenID

Deployment

Companies using each technology:

  • OpenID - Google, SixApart, Yahoo!, Flickr, MySpace, Facebook, Wordpress, Verisign, AOL, and many more...
  • OAuth - Google, Yahoo!, Twitter, OpenSocial, Netflix, Vimeo, and many more...
  • WebFinger - Google
  • .well-known - ???
  • WebID - spec isn't finished yet, but already several interoperable implementations.

WebID Enables

WebID is an enabling technology for...

  • Web payments and micro-financing - PaySwarm
  • Decentralized social networking - Identi.ca, Diaspora, etc.
  • eGovernment and verifiable online elections - Helios

Learn More about Identity on the Web

  • WebID
    • Decentralized Identity and Discovery for the Web.
  • OpenID
    • Open identity for the Web.
  • OAuth
    • Authorization of Web Agents to operate on one's behalf.
  • XAuth
    • Cross-site preferences cookie jar - solves NASCAR problem.
  • WebFinger
    • Discovery of identity information via e-mail addresses
  • /.well-known/ URIs
    • Directory for well-known files such as robots.txt, icon files, avatar images, etc.
  • Portable Contacts
    • Universal friends and contacts, remove friend entry repetition.
  • Activity Streams
    • A format for syndicating social activities around the Web.
  • Open Social
    • Open APIs for social network interoperability.
